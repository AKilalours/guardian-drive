00 — Overview

What is Guardian Drive?

Guardian Drive is a research prototype for a unified AI driver safety system. It bridges the domain gap that no commercial AV platform has closed: real-time physiological monitoring of the driver fused with bird's-eye-view scene perception, emergency dispatch, and live visualization — all in one pipeline.

One-line summary: Guardian Drive detects cardiac events, drowsiness, crashes, strokes, hypoglycemia, and seizures from physiological signals — then fuses that risk score with real BEV perception from 6 nuScenes cameras — and routes to the nearest real hospital or rest stop via live OSM, with voice dispatch and Twilio emergency contacts, displayed on a live Dreamview dashboard updating every 1.2 seconds.

What is real

  • PTB-XL ECG dataset for cardiac (18,869 records)
  • WESAD wearable stress & affect dataset
  • 404 real nuScenes Singapore camera frames
  • OpenDriveFM BEV inference 317fps
  • Real OSM routing — Skyway Motel, Mount Sinai West
  • Waymo Motion v1.3.1 — 262MB real validation data
  • Live webcam face detection (Task B, no --no-webcam)
  • Twilio/Discord emergency dispatch chain
  • WebSocket dashboard updating every window

What is simulated

  • ECG/EDA/IMU signals replayed from WESAD (no wearable hardware)
  • Crash IMU from scenario parameters (no real g-sensor)
  • Blood pressure from HRV proxy (no PPG sensor)
  • CARLA closed-loop (KinematicBicycle physics, not Unreal Engine)
  • Federated learning fleet (8 simulated vehicles, no real fleet)
  • Speed from mock GPS (no OBD-II)

01 — Architecture

Unified Pipeline

Five layers run in parallel every 1.2-second window — physiological signals, BEV perception, fusion engine, location response, and dashboard — connected via WebSocket like a human body to its brain.

01

SIGNAL ACQUISITION — Real WESAD + PTB-XL

Physiological signals (ECG, EDA, IMU, respiration) replayed from WESAD wearable dataset. PTB-XL provides 18,869 real clinical ECG records for cardiac screening. Signal Quality Index (SQI) computed every window from motion level and ECG quality fields. Driver baseline calibrated over first 5 windows.

ECG · EDA · IMU · RESP SQI 0.95 PTB-XL 18,869 records WESAD wearable dataset
02

PHYSIOLOGICAL TASKS A-H — 8 Parallel Detectors

Eight independent ML models run every window: cardiac arrhythmia (1D-CNN, AUC 0.961), 5-level drowsiness (TCN, AUC 0.951), crash detection (<0.15ms latency), stroke FAST screening (MediaPipe face mesh), pre-crash 30s horizon, cuffless blood pressure, hypoglycemia (EDA spike + 4-8Hz tremor), seizure (bilateral IMU + steering lockup). Each produces a risk score and label.

AUC 0.961 CARDIAC AUC 0.951 DROWSY g=9.2 CRASH 8 TASKS PARALLEL
03

KALMAN FUSION — Guardian Risk Score

Inverse-variance Kalman filter fuses all 8 task scores into a single Guardian Risk Score (GRS). Medical cross-task override: AFib + drowsiness simultaneously triggers hospital routing regardless of individual thresholds. State machine transitions: NOMINAL → ADVISORY → CAUTION → PULLOVER → ESCALATE with hysteresis and corroboration requirements.

GRS 0.882 CRASH_SEVERE GRS 0.672 DROWSY GRS 0.000 NORMAL 5-LEVEL STATE MACHINE
04

BEV PERCEPTION — Real nuScenes 6-Camera

OpenDriveFM runs on 404 real nuScenes validation samples (Singapore urban, scene-0655 and scene-1077). Six cameras (FRONT, F-LEFT, F-RIGHT, BACK, B-LEFT, B-RIGHT) feed a 128×128 BEV occupancy grid at 317fps. Guardian Drive BEV module produces 5 real agent detections per window (PED, MOT, CAR, BUS, TRU) with confidence scores and velocity arrows. TTC computed from trajectory vs occupancy intersection.

IoU 0.136 · ADE 2.457m 317 FPS 404 REAL FRAMES 5 AGENT CLASSES
05

LOCATION RESPONSE — Real OSM Routing

State-driven POI selection: FATIGUE→café within 1mi, DROWSY→motel within 3mi, PULLOVER→parking or motel within 5mi, ESCALATE→hospital only. Overpass OSM API returns real places: Skyway Motel 1.6mi, Mount Sinai West 3.0mi ETA 5.7min, Lenox Hill Hospital 4.3mi. Google Maps URLs generated. Voice messages contextual per state/level. Twilio SMS + Discord webhook fire on ESCALATE. 60s countdown timer at PULLOVER. Level 2 soft contact notification at CAUTION.

MOUNT SINAI WEST 3.0mi SKYWAY MOTEL 1.6mi REAL OSM ADDRESSES TWILIO + DISCORD
06

WEBSOCKET DASHBOARD — Live Dreamview

FastAPI backend pushes full payload every window via WebSocket. Dashboard updates BEV canvas, agent boxes, camera strip, risk gauge, task bars, module LEDs, POI card, voice output — all live with no page refresh. Web Audio API alarm sounds (single beep → double → triple → sawtooth siren by state level). JSONL event logging every window.

02 — Physiological Tasks

8 Detection Tasks

Eight independent real-time detectors covering the major medical causes of fatal driving incidents.

TASK A
Cardiac Arrhythmia
1D-CNN on PTB-XL 12-lead ECG. Detects AFib, tachycardia, bradycardia, heart block. RR irregularity + P-wave fraction + HR.
AUC 0.961 PTB-XL 18,869 FAR 0/hr
TASK B
Drowsiness / Fatigue
Temporal CNN on WESAD. 5 levels: ALERT → FATIGUE → DROWSY → MICROSLEEP → FULL_SLEEP. Circadian multiplier, driver baseline, EAR + PERCLOS from webcam.
AUC 0.951 5 LEVELS WESAD REAL
TASK C
Crash Detection
IMU g-peak threshold with jerk detection. MILD (2-4g), MODERATE (4-7g), SEVERE (>7g). Sub-millisecond latency. Bypasses state machine hysteresis — immediate ESCALATE.
g=9.2 SEVERE <0.15ms conf 0.98
TASK D
Stroke FAST Screening
MediaPipe 468-point face mesh. Detects facial asymmetry (>0.30), speech slur proxy (response latency), arm drift from steering. Multi-signal corroboration — single signal discounted 28%.
468 LANDMARKS FAST PROTOCOL MEDIAPIPE
TASK E
Pre-Crash Prediction
30-second horizon prediction. Fuses drowsiness trajectory, HRV trends, steering entropy, and BEV TTC to predict crash probability before it happens. NONE / LOW / MEDIUM / HIGH.
30s HORIZON MEDIUM 0.324 4 LEVELS
TASK F
Cuffless Blood Pressure
HRV proxy estimation from ECG RR intervals. Systolic/diastolic estimate with NORMAL/ELEVATED/STAGE1/STAGE2 classification. conf=0.45 (cuffless limitation noted).
118/74 NORMAL HRV PROXY conf 0.45
TASK G
Hypoglycemia Detection
EDA spike (sympathetic activation) + 4-8Hz IMU tremor (ictal frequency) + compensatory tachycardia + HRV suppression. Targets 100M+ diabetic drivers globally. Weighted fusion: EDA 35%, tremor 25%, HR 20%, HRV 15%, steering 5%.
SEVERE 0.84 4-8Hz TREMOR EDA SPIKE
TASK H
Seizure Detection
Bilateral 3-5Hz IMU tremor + steering lockup (std <0.005 for >2s) + ictal tachycardia. Detects tonic-clonic (0.77), absence (steering freeze), focal (unilateral). Time-to-lane-departure estimated from speed and lane width.
TONIC-CLONIC 0.77 TTLD 1.0s 3 TYPES
Medical cross-task override: AFib (Task A) + Drowsiness (Task B) firing simultaneously bypasses normal state machine progression and routes directly to hospital — not a rest stop. Crash (Task C) >7g bypasses all hysteresis immediately.

10 — Honest Gaps

Future Work

These 4 items are not built. They are correctly labeled as future work in the paper. All require hardware or institutional access not available during development.

Honest engineering: Every benchmark, metric, and result in this README is reproducible by running the listed command. Nothing is fabricated. The 4 items below are explicitly called out because they require hardware we don't have, not because we forgot to build them.

HARDWARE CARLA Real Server

CARLA 0.9.15 requires Linux with NVIDIA GPU. All download links are broken or S3-blocked from free platforms. Requires GCP VM ($0.45/hr) or local Linux GPU machine. Synthetic eval (KinematicBicycle) produces equivalent paper metrics.

DATASET nuPlan Closed-Loop

1500GB dataset. Requires motional academic registration and institutional server. OLS/NR-CLS/R-CLS/PDMS closed-loop scores would require 3-4 weeks of access and compute.

PHD LEVEL VLA Model

Vision-Language-Action model (camera → natural language → steering). Google DeepMind RT-2, Physical Intelligence, Tesla FSD teams have 200+ engineers on this. This is a PhD dissertation on its own.

HARDWARE openpilot CAN Bus

Requires physical car with OBD-II port and comma.ai hardware (~$200). opendbc has 200+ car CAN databases. Real steering/throttle/brake signals require the physical vehicle integration.
